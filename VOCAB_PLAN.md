## 一、两个项目的分工

| 项目 | 性质 | 单词系统职责 |
|------|------|------------|
| `english2-daily`（精读·田静每日一句） | 深、少而美、已讲透 | **复习**：已做独立单词页（见下「可复用资产」） |
| `reading`（本项目·历年考研英二真题 OCR） | 广、量大、真题语料 | **收集**：历年真题生词归一处、跨年去重排序 |

动机（唐迟）：**历年阅读里的单词会在多年后的真题上复现**，所以提前收集=提前见过。
最基础的词交给「不背单词」app 过，本系统只管中高阶、有语境的词。

## 二、reading 侧要做的事（收集管线）

数据源：`english2/ocr/*.md`（16 个年份的真题正文，已 OCR）。

1. **抽词**：从正文提取英文 token（注意剔除题号 `21.`、选项标 `[A]/[B]`、`&emsp;` 等 OCR 脚手架）。
2. **词形还原 lemmatize**：endured→endure、studies→study。无重依赖的做法：内置一份 lemma 映射表（开源 **AntBNC lemma list** 可下）。
3. **剥离基础词**（用户已定：**词频自动剥离**，不用 LLM 判级）：减去一份基础词表 → 剩下即候选生词。⚠️**外部依赖**：需引入基础词表（**NGSL ~2800** / 考研大纲 5500 / 高中 3500），本地暂无，需下载或生成。
4. **跨年复现计频**：统计每个候选词出现在 16 年中的几年 → 出现年数越多优先级越高（这步从现有 16 文件免费算，直接命中唐迟"复现"逻辑）。
5. **产出收集库**：每词带 出处年份 + 复现次数 + 所在句（语境）。形式建议先轻：`vocab_reading.md`（GitHub 可浏览），跑通后再考虑页面。

## 四、刻进项目的设计原则（直到考试，勿违）

- **只要学就当不会**：不自评"是否认识"（噱头），不宣称学会，只管反复见、反复读。
- **复习=读句子/用起来**：把词连同句子直接端到页面读它、用它、感受它，不做判断按钮。
- **工具克制**：一屏即收手（像 cat / 日料一份），不堆功能；不同关注点拆**不同页面**，别都塞一处。
- **工程口味**：单一职责模块、防重复 token、命令优雅、视觉简洁。

## 五、可复用资产（在 `../english2-daily`，按范式照搬，别盲目复制以免重复）

- `scripts/vocab.py` — 词库解析的**单一职责模块范式**（reading 源是真题正文、逻辑要另写，但结构/接口照此：一个 `collect_*` 库函数喂所有视图 + 一个 CLI 产 md）。
- `scripts/word.py` + `word.html` + `word.sh` — **读式复习页范式**：一屏 10 词、词义与句子中文用**胶带**遮住、口译过一遍再揭开对一眼；调度=**读式再曝光**（读过一次按 1/2/4/7/12/20/30/45/60 天渐宽再现、**永不毕业**），状态 `data/vocab.json` 本机本地不入 git。reading 的词若日后也要复习，直接复用这套 UI 与调度。
- `scripts/studio.py` — **本地/远程双模式**范式（扫 /proc 判定有无 daemon → 本地写文件唤醒 / 远程 commit+push 经 GitHub 同步）。
- `scripts/scores.py` — CLI 产 md + 库函数复用 的同构样板。
